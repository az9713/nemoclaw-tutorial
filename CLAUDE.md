# CLAUDE.md — NemoClaw Development Guide for Claude Code

## Project Overview

NemoClaw is an NVIDIA open-source project that runs OpenClaw (AI agent framework) inside OpenShell (secure sandboxed runtime) with NVIDIA inference routing. It has a two-layer architecture: a Node.js host CLI and a TypeScript OpenClaw plugin, connected by a Python blueprint orchestrator.

## Repository Structure

```
bin/nemoclaw.js          — Host CLI entry point (Node.js, CommonJS)
bin/lib/                 — CLI modules (runner, registry, credentials, onboard, nim, policies, preflight)
nemoclaw/src/            — TypeScript plugin source (ESM, strict mode)
nemoclaw/src/index.ts    — Plugin registration entry point
nemoclaw/src/cli.ts      — Commander.js subcommand wiring
nemoclaw/src/commands/   — Command handlers (launch, migrate, connect, status, logs, eject, onboard, slash)
nemoclaw/src/blueprint/  — Blueprint lifecycle (resolve, fetch, verify, exec, state)
nemoclaw/src/onboard/    — Onboarding config (config, prompt, validate)
nemoclaw/dist/           — Compiled JS output (DO NOT edit, generated by tsc)
nemoclaw-blueprint/      — Python blueprint + security policies
test/                    — Node.js native test runner tests
scripts/                 — Shell scripts for setup/deployment
Dockerfile               — Sandbox container image
```

## Build Commands

```bash
# Build TypeScript plugin
cd nemoclaw && npm run build

# Run tests
npm test

# Lint
cd nemoclaw && npm run lint

# Format check
cd nemoclaw && npm run format:check

# Full check (lint + format + type check)
cd nemoclaw && npm run check

# Clean build artifacts
cd nemoclaw && npm run clean
```

## Key Technical Details

- **Node.js**: Minimum v20.0.0
- **TypeScript**: Strict mode, ESM output, compiled to `dist/`
- **Host CLI** (`bin/`): CommonJS, no build step needed
- **Plugin** (`nemoclaw/src/`): TypeScript with ESM imports (use `.js` extensions in imports)
- **Blueprint runner**: Python 3.11+, called as subprocess from TypeScript
- **Tests**: Node.js built-in test runner (`node --test`), no Jest/Mocha
- **No database**: All state stored in JSON files under `~/.nemoclaw/`

## Coding Conventions

- Apache-2.0 SPDX header on all source files
- TypeScript strict mode with no `any` types where avoidable
- Commander.js for CLI argument parsing
- Shell commands via `execSync`/`execFileSync` (never `shell: true` in Python runner)
- Error handling: log + return early pattern, no thrown exceptions in command handlers
- Security: credentials stored with mode 0o600, no API keys in code or logs

## State Files

- `~/.nemoclaw/state/nemoclaw.json` — Plugin run state
- `~/.nemoclaw/config.json` — Onboard inference config
- `~/.nemoclaw/credentials.json` — API keys (mode 600)
- `~/.nemoclaw/sandboxes.json` — Sandbox registry
- `~/.nemoclaw/blueprints/<ver>/` — Cached blueprints

## Communication Protocol

The TypeScript plugin calls the Python blueprint runner as a subprocess. The runner communicates back via stdout:
- `PROGRESS:<0-100>:<label>` — Progress updates
- `RUN_ID:<id>` — Run identifier
- Exit code 0 = success, non-zero = failure

## Common Patterns

- Blueprint lifecycle: resolve → verify → plan → apply
- Migration: detect host → snapshot → archive → sync → verify → save state
- Rollback: load state → blueprint rollback → restore snapshot → clear state
- Onboard: preflight → gateway → sandbox → NIM → inference → OpenClaw → policies
